# 🎨 Viled Fashion Design Tokens

Репозиторий содержит токены дизайн-системы **Viled Fashion**.
Это единый источник визуальных параметров для всех цифровых продуктов: web, iOS, Android.

---

## 📂 Структура

```
Viled/
  Primitive.json
  Core.json
  Global.json
  Semantic.json
  $themes.json
```

### Primitive

Базовые атомарные значения без контекста:

* raw цвета;
* базовые размеры;
* прозрачности;
* фундаментальные параметры.

Не используются напрямую в интерфейсах.

---

### Core

Системные токены:

* типографика;
* spacing;
* grid;
* radius;
* elevation;
* motion.

Формируют основу дизайн-системы.

---

### Global

Брендовые токены:

* цветовые палитры;
* глобальные настройки бренда;
* темы.

---

### Semantic

Смысловые токены:

* background;
* text;
* border;
* icon;
* state;
* status (success, error, warning).

Используются в UI и коде.

---

### Themes

Темы:

* light;
* dark;
* кампании и сезонные.

---

## ⚙️ Инструменты

* Figma
* Token Studio
* Git
* CI/CD экспорт в кодовые базы.

# 👩‍🎨 Мануал для дизайнеров

## 1. Основной принцип

Работа ведётся только через **Semantic токены**.
Primitive и Core менять без согласования нельзя.

---

## 2. Как изменить токен

1. Открыть Figma → Token Studio.
2. Найти semantic токен.
3. Изменить значение.
4. Проверить:

   * light / dark;
   * основные экраны;
   * компоненты.
5. Commit → Push в Git.

---

## 3. Когда создавать новый токен

Создавайте, если:

* значение повторяется;
* есть бизнес-смысл;
* требуется масштабируемость.

Не создавайте:

* для одного случая;
* если можно использовать существующий.

---

## 4. Где создавать

| Задача            | Слой     |
| ----------------- | -------- |
| Новый цвет бренда | Global   |
| Новый spacing     | Core     |
| UI-цвет           | Semantic |

---

## 5. Нейминг

Формат:

```
category.group.state
```

Пример:

```
background.primary.default
background.primary.hover
text.secondary
border.focus
```

---

## 6. Запрещено

* менять Primitive;
* удалять токены;
* создавать дубликаты;
* использовать raw значения.

---

# 👨‍💻 Мануал для разработчиков

## 1. Основной принцип

Используются только **Semantic токены**.
Core и Primitive — технический слой.

---

## 2. Подключение

Токены автоматически экспортируются в:

* CSS variables;
* iOS;
* Android.

Пример:

```css
color: var(--text-primary);
background: var(--background-primary);
```

---

## 3. Темы

Переключение темы не требует изменений в коде.
Все значения управляются через `$themes.json`.

---

## 4. Breaking changes

Возможны при:

* изменении semantic токенов;    
* удалении токенов;
* изменении структуры.

Следите за changelog.

---

## 5. Если токена нет

1. Не использовать hardcode.
2. Создать issue.
3. Описать:

   * сценарий;
   * платформу;
   * частоту использования.

---

## 6. Версионирование

Semantic versioning:

* major — breaking;
* minor — новые токены;
* patch — фикс.

---

## 🤝 Workflow

1. Дизайнер меняет токен.
2. PR.
3. Проверка Design System.
4. Merge.
5. Авто-обновление.
